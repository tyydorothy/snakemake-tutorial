### How Snakemake Handles Wildcards

---

### What is a wildcard?

A wildcard is a **placeholder** in file paths that Snakemake fills in automatically by pattern matching.

```python
output: "data/samples/{sample}.fastq"
```

`{sample}` is the wildcard — it can match any value.

---

### How Snakemake infers wildcard values:

Snakemake works **backwards from the target file** you request:

```
You request: data/samples/A.fastq
                           │
Snakemake matches against: data/samples/{sample}.fastq
                           │
Infers:  sample = "A"
```

---

### Where the inferred value is stored:

Snakemake stores it in a **`wildcards` object** for that specific job:
```python
wildcards.sample = "A"
```

This object is then accessible in:
- `input` paths → `{sample}` (shorthand, Snakemake expands automatically)
- `output` paths → `{sample}` (shorthand)
- `shell` commands → must use `{wildcards.sample}` explicitly

---

### Why shell needs `{wildcards.sample}` explicitly:

Inside `input`/`output`, Snakemake knows `{sample}` refers to the wildcard automatically. But inside `shell`, it's just a Python format string — Snakemake can't assume `{sample}` means a wildcard, so you must be explicit:

```python
rule gen_random_fastq:
    output:
        "data/samples/{sample}.fastq"   # {sample} auto-resolved ✅
    shell:
        "echo {sample}"                 # ❌ NameError - unknown name
        "echo {wildcards.sample}"       # ✅ explicitly access wildcard object
        "echo {output}"                 # ✅ best practice - already contains sample name
```

---

### Full picture:

```
You run: snakemake data/samples/A.fastq
                │
                ▼
Snakemake matches output pattern: "data/samples/{sample}.fastq"
                │
                ▼
Infers: wildcards.sample = "A"
                │
                ▼
Fills in input:  "data/genome.fa"        (no wildcard here)
Fills in output: "data/samples/A.fastq"  ({sample} → "A")
Fills in shell:  needs {wildcards.sample} to access "A"
```

---

### TL;DR

| | How to access wildcard value |
|---|---|
| `input`/`output` | `{sample}` — Snakemake resolves automatically |
| `shell` | `{wildcards.sample}` — must be explicit |
| Best practice in shell | use `{output}` or `{input}` instead — already expanded |
